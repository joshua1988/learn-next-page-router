# 1. 테스팅 소개

---

## 왜 테스트 코드가 필요할까?

- 모든 시스템을 일일이 직접 테스트하는 건 불가능
- 규모가 작은 시스템은 필요 없을수도 있다
- 규모가 커질수록 테스트할 수 있는 범위의 한계와 테스트 시간이 늘어난다

---

## 무엇을 테스트 해야할까?

사용자 관점의 인터렉션과 기능 동작

---

## 어떻게 테스트 해야할까?

대중적인 라이브러리들을 활용할 것

예) React Testing Library, Cypress
